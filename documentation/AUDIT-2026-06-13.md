## 1. Architecture & Folder Structure

Layout is **standard, idiomatic Strapi 5** — `src/api/<feature>/{content-types,controllers,routes,services,graphql}` is correct and scalable. The real problems are at the config layer, not the structure.

### 🔴 S1 — Production overrides silently dead
Strapi loads environment config from `config/env/<environment>/`. The files live at `config/env/production/`**`production`**`/…` — one level too deep. **None load in production.**

Consequence: the hardened production `middlewares.ts` (tighter CSP, no `unsafe-inline`), production `server.ts` (`RENDER_EXTERNAL_URL` proxy URL), `database.ts`, `logger.ts`, `cron-tasks.ts`, `plugins.ts` are all ignored. Production silently runs the looser base `config/middlewares.ts`.

```
# Fix — flatten one level
config/env/production/production/middlewares.ts  →  config/env/production/middlewares.ts
# (repeat for server.ts, database.ts, logger.ts, plugins.ts, cron-tasks.ts)
```

### 🟡 Recommendations
- Extract shared form logic out of per-API folders into reusable units — `src/utils/recaptcha.ts` and `src/utils/sendTemplatedEmail.ts` (see C1).
- Delete the empty `config/security.ts` dead file.

## 2. Feature Implementation Quality

### 🔴 S2 — reCAPTCHA bypass via REST
Each form uses `factories.createCoreRouter(...)`, which exposes default REST CRUD including `POST /api/form-contacts`. reCAPTCHA + required-field checks live **only** in the GraphQL resolver.

- If the Public role has `create` enabled for the frontend (or anyone enables it), the REST route writes records and triggers `afterCreate` emails with **zero captcha** → spam / email-bomb amplification.
- If Public `find`/`findOne` is on, all submitted PII (name/email/phone/message) is publicly readable.

**Mitigation:** disable REST for these collections (custom `routes` with `only: []`, or route-level `policies`), and move captcha + validation into the content-type lifecycle or a policy so it runs regardless of transport. Audit the Public role (Settings → Roles) for these 3 collections; ensure `find` is OFF.

### 🟡 S7 — Silent failures
`afterCreate` wraps everything in `try { … } catch (err) { console.error(...) }`. Email failure → user still gets a success response, `emailSent` stays `false`, no retry. The `if (!result.emailSent)` guard is also moot: `afterCreate` fires once per create and the record is fresh, so it never dedupes anything real.

### 🟡 Logic bugs in `form-reservation-tour/.../lifecycles.ts`
- `.replace('{{ article }}', result.article)` → injects `"[object Object]"` (it's a relation object).
- `result.people` / `result.tourDate` interpolated without guards → `"undefined"` if absent.
- `String.replace(string, …)` replaces **only the first** occurrence — duplicate placeholders in a template won't all fill. Use `replaceAll` or a `{ key: value }` reduce.

## 3. Code Quality & Anti-patterns

### 🔴 C2 — Shipped boilerplate
`src/api/form-reservation-tour/content-types/form-reservation-tour/lifecycles.ts:39-40`:

```ts
from: 'noreply@mainstylis.com',                          // wrong domain (not suyai); likely unverified in SendGrid → silent fail
subject: 'The Strapi Email plugin worked successfully',  // template leftover sent to real customers
```

The other two forms correctly use `env.SMTP_EMAIL_ADMIN`. This one almost certainly **never delivers** (unverified sender → caught & swallowed by S7).

### Other smells
- `readFileSync` on every submission — blocking I/O on the event loop; templates should be read once / cached.
- 6-level relative path `'../../../../../../config/email/templates/...'` via `__dirname` — brittle; depth differs between `src/` and compiled `dist/`, and `config/` isn't guaranteed in the build output. Use `strapi.dirs.app.root`.
- `draftAndPublish: true` on form-submission collections — submissions become *drafts*; wrong model for write-only intake.
- Inconsistent schema: `message` maxLength 500 (contact) vs 400 (tour); tour fields lack `required` (only enforced in resolver).

## 4. Refactoring Opportunities (DRY / SOLID)

### 🔴 C1 — Eliminate triplication
`services/recaptcha.ts` is byte-identical across all 3 APIs, and the GraphQL resolvers + lifecycle email logic are ~90% duplicated. Single source of truth:

**Before** — 3 copies of `recaptcha.ts`, 3 near-identical resolvers, 3 near-identical lifecycles.

**After** — one util + one factory:

```ts
// src/utils/recaptcha.ts  (single copy)
export async function validateRecaptcha(token: string, action: string): Promise<boolean> { /* … */ }
```

```ts
// src/utils/createFormResolver.ts
import { validateRecaptcha } from './recaptcha';

export const createFormResolver = (uid: string, required: string[], action: string) =>
  async (_p: unknown, args: { data: Record<string, unknown> & { recaptchaToken?: string } }) => {
    const { recaptchaToken, ...formData } = args.data;
    if (!recaptchaToken) throw new Error('reCAPTCHA token is required');
    if (!(await validateRecaptcha(recaptchaToken, action))) throw new Error('reCAPTCHA inválido.');
    for (const f of required) if (!formData[f]) throw new Error(`El campo ${f} es requerido.`);
    return strapi.documents(uid as any).create({ data: formData });
  };
```

```ts
// src/utils/sendTemplatedEmail.ts — escapes input, caches templates, async read
import fs from 'fs';
import path from 'path';
import { escape } from 'lodash';

const cache = new Map<string, string>();

export async function renderTemplate(name: string, vars: Record<string, unknown>) {
  const file = path.join(strapi.dirs.app.root, 'config/email/templates', `${name}.html`);
  if (!cache.has(file)) cache.set(file, await fs.promises.readFile(file, 'utf-8'));
  return Object.entries(vars).reduce(
    (html, [k, v]) => html.replaceAll(`{{ ${k} }}`, escape(String(v ?? ''))),  // ← HTML-escape fixes S3
    cache.get(file)!,
  );
}
```

Each `lifecycles.ts` collapses to a few lines calling `renderTemplate(...)`. The action name `'tourReservation'` is currently hardcoded for **all three** forms (incl. contact) — parameterize it.

## 5. Security Audit

### 🔴 S3 — HTML / email injection (XSS-in-email)
Every `lifecycles.ts` does `template.replace('{{ name }}', result.name)` with **no escaping**. Attacker submits `name = <a href=evil>…</a>` (or `<img src=x onerror=…>`); it is injected raw into HTML emailed to **both the customer and the admin copy inbox** → phishing / content-spoofing from your own domain. Fix: HTML-escape all interpolated values (see C1's `escape(...)`).

### 🟡 S4 — GraphQL exposure
`config/plugins.ts`: `playground: true` and `introspection: true` are **hardcoded** (only `landingPage` is env-gated). In production this leaks the full schema and an interactive console. With `maxLimit: 500` and `depthLimit: 7`, that's an enumeration + query-cost/DoS surface. Gate both on `env('NODE_ENV') !== 'production'`.

### 🟡 S5 — Mass assignment
Resolvers spread `...formData` straight into `create`. Caller can set `emailSent: true` (skip the email) or attach an arbitrary `article` relation id. Whitelist fields explicitly instead of spreading.

### 🟡 S6 — Deploy mismatch
`render.yaml` → `NODE_VERSION: ~16.13.0`; `package.json` engines require `>=18`, and Strapi 5 needs Node 18/20/22. Build/runtime will fail or be unsupported. Bump to `20.x`.

### 🟢 Good / verified
- Secrets clean in git — `.env`, `*.sql`, `*.tar/.gz` are gitignored; only `.env.example` (placeholders) is tracked.
- reCAPTCHA threshold `score >= 0.5 && tokenProperties.valid` is reasonable; fails closed on errors.
- CORS is env-driven (`FRONTEND_URL`) in base config.
- Cloudinary / SendGrid creds via `env()`.

### 🟢 Housekeeping (not in git, but on disk)
`strapi_backup_old.sql` (431 KB), `export_*.tar.gz`, and `data/strapi - Copy.sqlite` sit in the repo working tree. Move them out of the project dir — they are easy to leak via a future `git add -f` or a misconfigured static mount.

## Recommended fix order

1. **S1** — flatten `config/env/production/` (restores all prod hardening at once).
2. **S2** — lock down REST routes + audit Public role permissions.
3. **S3** — HTML-escape email interpolation.
4. **C2** — fix the `mainstylis.com` / test-subject leftover.
5. **S4 / S5 / S6** — env-gate GraphQL, whitelist fields, bump Node.
6. **C1** — consolidate the triplicated code.
